# AIへの指示：クラウドコスト管理とタグ付けに関する原則

AIにコンテンツ（記事、説明文、ドキュメントなど）を作成させる際、以下の「Wixのコスト削減事例」に基づく原則を意識させてください。

## 1. 核心となる問題：コストの「ブラックボックス」化

* **問題点:** クラウド（特にデータプラットフォーム）は、多くのチームが共有で利用するため、「誰が」「何の目的で」「どれだけ」リソースを使っているかが不明瞭になりがちである。
* **状態:** コストが「どんぶり勘定」になり、無駄なリソースが放置されていても誰も気づけない「ブラックボックス」状態に陥っていた。

## 2. 解決策：標準化されたタグ付け (Standardized Tagging)

* **目的:** コストが発生するすべてのインフラリソース（仮想マシン、ストレージ、データベース等）に、**統一されたルールでラベル（タグ）を付ける**ことを徹底する。
* **役割:** タグは、技術的なリソースを「ビジネス的な単位」に紐付けるためのメタデータとして機能する。

### タグ付けの具体例

タグは以下のような「キー：バリュー」形式で付与する。

```

team: analytics
project: billing-system
env: production
owner: richard@example.com

```

## 3. 最も重要な概念：「利用状況の帰属 (Usage Attribution)」

* **定義:** 「タグ付け」によって、**すべてのコストの「持ち主」を明確にすること**。
* **効果:** これにより、技術的なコスト（サーバーAの利用料）を、ビジネス上のコスト（「分析チーム」の「課金システムプロジェクト」のコスト）として集計できるようになる。

## 4. 期待される成果（コスト削減への道筋）

「利用状況の帰属」が可能になることで、以下の好循環が生まれる。

1.  **可視化 (Visibility):**
    * どのチームが、どのプロジェクトで、どれだけコストを使っているかが一目でわかるようになる。
2.  **最適化 (Optimization):**
    * 「このプロジェクトは終わったのに、リソースが起動したままだ」といった**具体的な無駄**を発見し、削減できる。
3.  **説明責任 (Accountability):**
    * 各チームが自分たちの利用コストを把握し、コスト意識を持つようになる（「自分ごと」化）。

このプロセス全体が、Wixが達成したような大幅なコスト削減（事例では50%）につながる。
